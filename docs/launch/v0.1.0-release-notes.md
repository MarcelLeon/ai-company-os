# AI Company OS v0.1.0 — Public Launch Release Notes

> Draft. Move to GitHub Release body when you cut the v0.1.0 tag.

## What's in 0.1.0

The first public, dogfooded release of AICO. Three months of `/overnight`
runs against this very repository proved out the absence-first loop.

### Features

- **IM-first orchestration** — Telegram primary control plane; Feishu
  (text + webhook + idempotency) MVP for the second channel.
- **Six adapters** — Claude Code (full session resume), Codex (read-only
  by default), Cursor, CodeFlicker, Trae, Gemini. All opt-in. All under
  the same risk + approval matrix.
- **Project office model** — `/project`, `/team`, `/roles`, `/appoint`,
  `/lead`, `/ask`, `/brief`, `/risks`, `/blockers`, `/next`, `/daily`,
  `/weekly`.
- **Safety + ops** — `/approve`, `/reject`, `/interrupt`, `/tasks`,
  `/task`, `/metrics`, `/audit`, plus boss-only `/undo` and `/why`,
  plus lead-only `/timeline` and `/rollback`.
- **A2A memory fabric** — `/remember`, `/recall`, `/forget`,
  `/experience` (candidate → active), semantic + graph retrieval, scope
  governor (boss / project / team / role / agent), purpose tags.
- **Absence loop** — `/overnight`, `/morning`, `/inbox`, `/dream`,
  Outcome Grader, `/goal` Goal Brief, handoff completeness guard.
- **aico-view** — read-only mobile web with token auth, plus
  IM-delivered self-contained HTML snapshot via `/view`.
- **Restart-aware state** — SQLite task / approval / delegation store
  plus append-only JSONL audit and memory ledgers.

### Engineering quality

- 428 unit tests passing, 1 skipped.
- `mypy --strict`, `ruff check`, `ruff format --check` all green in CI.
- 36 ADRs documenting accepted architectural decisions.
- `docs/journal/PITFALLS.md` indexed through P-038.
- 146 documented development rounds in `docs/journal/ROUNDS.md`,
  including rejected alternatives — the project is intentionally
  unusually transparent about *why* it doesn't do certain things.

### What this release explicitly does NOT do

- Not a sandbox. AICO is a control layer in front of local CLIs.
- Not an LLM provider. Bring your own Claude / Codex / Cursor / etc.
- Not a chat UI. The CLI exists for ops; everyday use is in IM.
- Not a multi-agent framework. The wedge is operations, not authoring.

### Try it in 30 seconds, no tokens

```bash
git clone https://github.com/MarcelLeon/ai-company-os.git
cd ai-company-os
env UV_CACHE_DIR=/tmp/aico-uv-cache uv run --python 3.11 aico-release-room-demo
```

Runs the Release Room flow end-to-end with deterministic fake adapters.
No Telegram bot. No Claude / OpenAI account. No spend.

### Wire it to a real Telegram bot + Claude Code

See [Quickstart](docs/human/quickstart.md). Minimum env:

```bash
export AICO_TELEGRAM_BOT_TOKEN=...
export AICO_CLAUDE_WORKING_DIRECTORY=$PWD
export AICO_PROJECT_CONFIG_PATH=config/projects.example.json
export AICO_PERSONA_CONFIG_PATH=config/personas.example.json
export AICO_AUDIT_LOG_PATH=/tmp/aico-audit.jsonl
export AICO_MEMORY_PATH=/tmp/aico-memory.jsonl
export AICO_STATE_DB_PATH=/tmp/aico-state.db
uv run aico-phase1
```
